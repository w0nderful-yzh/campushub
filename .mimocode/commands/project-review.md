# Project Review & Plan Update

Review the current project and create/update a development plan.

## Instructions

1. **Explore the codebase structure**
   - List directory structure
   - Identify tech stack (frontend/backend frameworks, database, etc.)
   - Check package.json/requirements.txt/pom.xml for dependencies

2. **Analyze current state**
   - What features are implemented?
   - What's working vs broken?
   - Code quality issues?
   - Missing tests?

3. **Identify improvements**
   - High priority: Bugs, missing core features
   - Medium priority: Code quality, performance
   - Low priority: Nice-to-haves, polish

4. **Create/update plan**
   - Write to `PLAN.md` or `plan.md` in project root
   - Use markdown checklist format
   - Group by priority (P0/P1/P2)
   - Include estimated effort

5. **Report findings**
   - Summary of current state
   - Key issues found
   - Recommended next steps

## Output

- Updated `PLAN.md` with prioritized tasks
- Brief summary to user of findings and recommendations
